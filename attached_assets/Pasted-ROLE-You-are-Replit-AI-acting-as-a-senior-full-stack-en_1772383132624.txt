PART B — BACKEND APIs (Node/Express)
========================================
Implement API endpoints. Keep consistent with existing auth patterns.

1) GET settings
GET /api/videos/:id/player-settings
Returns:
- playerSettings object (including logo/overlay/qr/clips/style + existing controls)
- banners array (sorted)
- resolved asset URLs (logoUrl, overlayUrl, introUrl, outroUrl) if possible

2) PATCH settings (atomic updates)
PATCH /api/videos/:id/player-settings
Body can include any subset of:
- logoEnabled, logoPlacement, logoSizePercent, logoOpacity, logoAssetId
- overlayEnabled, overlayOpacity, overlayMode, overlayAssetId
- qrEnabled, qrTitle, qrUrl, qrPlacement, qrSizePercent, qrOpacity, qrBgEnabled, qrBgOpacity
- introAssetId, outroAssetId, loopEnabled, loopMode
- brandColor, theme, showDisplayNames, showHeadlines, fontFamily
Validate all inputs.

3) Upload endpoints (MUST upload to Backblaze B2 / S3 bucket)
Use existing storage service and connections.

Logo upload:
POST /api/videos/:id/player-assets/logo
FormData: file
- Validate: image/png, image/webp, image/jpeg, max 20MB
- Upload to bucket path: assets/videos/:id/logo/<timestamp>_<filename>
- Create asset record
- Update playerSettings.logoAssetId and logoEnabled=true
Return: assetId + url

Overlay upload:
POST /api/videos/:id/player-assets/overlay
- Validate: image/*, max 20MB
- Recommended 1280x720, but don’t hard fail; warn if not.
- Upload to: assets/videos/:id/overlay/...
- Update overlayAssetId and overlayEnabled=true

Intro upload:
POST /api/videos/:id/player-assets/intro
- Validate: video/mp4, max 200MB
- Upload: assets/videos/:id/intro/...
- Optional: check duration <= 10 minutes using ffprobe if you already have it (otherwise skip duration check but keep max size)
- Update introAssetId

Outro upload:
POST /api/videos/:id/player-assets/outro
- same as intro

4) Banner CRUD
GET /api/videos/:id/banners
POST /api/videos/:id/banners
PATCH /api/videos/:id/banners/:bannerId
DELETE /api/videos/:id/banners/:bannerId
Also endpoint to reorder:
POST /api/videos/:id/banners/reorder  { orderedIds: string[] }

5) Security
- Ensure only authorized admin can update these settings (same auth you use for video edits).
- Use existing token/session.
