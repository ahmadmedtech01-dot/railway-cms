renewPlaybackSession({

  publicId,

This must:

- validate LMS origin/client if LMS
- validate share link if share
- extend/rotate token
- return new stream window and expiry info
- preserve security settings

LMS must not use a weaker or separate renewal path.

Your LMS sessions must update:
- watch time
- completion %
- last ping
- currentTime
- status

If LMS session shows active but watch time remains 0s, server-gated playlist logic may think user has not watched and reject chunks. Fix heartbeat/progress mapping for LMS sessions.
