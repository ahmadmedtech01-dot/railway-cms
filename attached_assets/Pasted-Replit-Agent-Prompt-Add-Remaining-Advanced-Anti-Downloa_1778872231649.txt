Required features:

1. Stealth Protected Playback Mode

Add/complete a mode that hides obvious HLS names from browser Network.

In this mode, Network should NOT show:
- master.m3u8
- index.m3u8
- .m3u8
- .ts
- .m4s
- seg_000.ts
- seg_001.ts
- /key

Use opaque player routes instead:
- /api/player/:publicId/stream/bootstrap
- /api/player/:publicId/stream/window
- /api/player/:publicId/stream/chunk/:opaqueChunkId
- /api/player/:publicId/stream/secret/:opaqueKeyId

Internally map opaque chunk IDs to real HLS segments.
Do not expose real storage URLs or real segment filenames.

If hls.js still needs a playlist:
- generate the playlist in memory as a Blob URL
- segment URLs inside it must be opaque signed chunk URLs
- visible Network must only show opaque stream/chunk/secret routes

2. MediaSource / AppendBuffer Guard

Create or complete:

client/src/lib/security/mediaSourceGuard.ts

Detect tampering/proxying of:
- window.MediaSource
- MediaSource.prototype.addSourceBuffer
- MediaSource.prototype.endOfStream
- SourceBuffer.prototype.appendBuffer
- SourceBuffer.prototype.remove
- URL.createObjectURL
- URL.revokeObjectURL

Capture native references before hls.js starts.
Recheck:
- before HLS attach
- after manifest parsed
- after first segment append
- every 10–15 seconds during playback

If tamper is detected:
- pause video immediately
- show “Security Risk Detected”
- send security event to backend
- increase abuse score
- revoke session after repeated/critical detection
- all stream/chunk/secret routes and legacy /hls /seg /key routes must return 403 after revocation

Events:
- MEDIA_SOURCE_HOOK_DETECTED
- APPEND_BUFFER_HOOK_DETECTED
- HLS_BUFFER_CAPTURE_SUSPECTED

Do not break normal hls.js playback.

3. Server-Gated Playlist Window

Fix playlist walking.

Playlist/window must NOT advance because segments are fetched.
It must advance only from backend-verified heartbeat currentTime.

Backend session should track:
- verifiedCurrentTime
- watchedSeconds
- lastHeartbeatAt
- maxAllowedSegmentIndex
- downloadedSegmentDuration
- downloadAheadSeconds
- lastAllowedWindowStart
- lastAllowedWindowEnd

Rules:
- only expose current small segment window
- default window size: 6 segments
- default max prebuffer: 30 seconds
- reject segment/chunk if ahead of verified playback window
- reject if heartbeat is stale
- reject if session is revoked or suspicious

Events:
- OUT_OF_WINDOW_SEGMENT_REQUEST
- OUT_OF_WINDOW_CHUNK_REQUEST
- SEGMENT_REJECTED
- CHUNK_REJECTED

4. Download Velocity Scoring

Add strict server-side scoring.

Track per session:
- playlist/window fetch rate
- segment/chunk fetch rate
- key/secret fetch rate
- downloadedMediaSeconds
- verifiedWatchedSeconds
- downloadAheadSeconds
- duplicate segment/chunk requests
- out-of-window requests

Default:
- maxDownloadAheadSeconds: 30–45 seconds
- heartbeat interval: 10–15 seconds
- suspicious threshold revokes session

If user downloads faster than real playback:
- add abuse score
- log DOWNLOAD_AHEAD_LIMIT_EXCEEDED
- revoke session when threshold is crossed
- return 403/429 from media routes

5. Short Token TTLs

Tokens must be:

- video-bound
- path-bound
- segment/chunk-index-bound
- quality-bound if applicable
- expiry-bound

Make TTLs configurable globally and per video.

7. Strong Heartbeat v2

- publicId

- playing/paused

- stop advancing playlist/window
- reject future chunk/key requests after grace period
