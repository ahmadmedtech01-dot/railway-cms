PART B — BACKEND API
========================================
1) Upload thumbnail
POST /api/videos/:id/thumbnail
- Auth: admin only (same as video edit)
- Input: multipart/form-data file
- Validate:
  - mimeType: image/png, image/jpeg, image/webp
  - max size: 5MB (or 10MB if you prefer; but enforce a limit)
  - recommended aspect ratio: 16:9 (1280x720). If not 16:9, still allow but warn in response.
- Upload to Backblaze B2/S3 using existing storage connection logic:
  path: assets/videos/:id/thumbnail/<timestamp>_<originalname>
- Create asset record (id, bucket, key, size, mime)
- Update video.thumbnailAssetId = asset.id
- Return:
  { thumbnailAssetId, thumbnailUrl }  // url must be usable by browser (public or signed)

2) Remove thumbnail
DELETE /api/videos/:id/thumbnail
- Set thumbnailAssetId = null
- (Optional) keep old asset in bucket (don’t hard delete) OR delete if your app does that elsewhere.

3) Ensure GET /api/videos/:id and list endpoints include thumbnailUrl
- If using signed URLs, generate a signed URL for the thumbnail on each response.
- If public, return public URL.
